# ArmStudio · 通用机器人控制与轨迹设计软件

一个纯前端、开箱即用的机器人仿真与轨迹设计工具。支持**任意结构的机器人**（机械臂、四足、SCARA…），可**导入自有机型**、在软件里**拖拽搭建机器人**、用**全身姿态关键帧**设计运动，并一键导出 **Arduino / ROS** 代码。

## 快速开始

用浏览器打开 `index.html` 即可（无需安装，推荐 Chrome / Edge，首次需联网加载 Three.js）。
进入时会弹出引导，可按当前机型一键加载示例。

## 核心功能

**多机型 & 导入**
- 内置：6 轴机械臂、四足机器人、SCARA 平面臂。左上角下拉切换。
- 导入：支持自定义 JSON 机型，以及**简化 URDF**（解析 link/joint/origin/axis/limit，自动 Z-up→Y-up 换算）。
- 导出：把当前机型（含改动）保存为 JSON。

**主流工业轨迹设计（MoveJ / MoveL / MoveC）**
- 每段运动可选三种主流指令：`MoveJ`(PTP 关节插补，路径最快)、`MoveL`(末端走直线)、`MoveC`(末端走圆弧，弧高可调)。
- 每段独立设定**速度 (mm/s)** 与**加速度**，运动按**梯形速度曲线**做加减速，更贴近真机。
- 全局**速度倍率**滑块整体调速。
- **TCP 轨迹可视化**：3D / 2D 中显示末端实际走过的笛卡尔路径，并按瞬时速度**色阶着色**（蓝慢→橙快）；直线段是直线、圆弧段是弧线、PTP 段是关节插补的真实曲线。
- 自动估算**循环节拍（cycle time）**。

**拖拽设计运动（全身姿态关键帧）**
- 3D 视图：拖动末端执行器球体，自动逆解；多末端机型（如四足 4 个足端）可在右侧选择当前末端。
- 2D 俯视图：点击 / 拖动设定当前末端的 X/Z，配合高度滑块设定 Y。
- **目标位置面板**：直接键入末端目标 X/Y/Z 坐标并「➤ 移动到该点」；并可设「到点速度」（mm/s），捕获关键帧时自动写入该段。
- **🖱 点击取点模式**：动作模式下一键开启，在 3D 场景点击地面平面即可连续落下运动点——每次点击自动逆解到该点并捕获一个关键帧（高度取自 2D 高度滑块、速度取自"到点速度"）；拖动鼠标仍可旋转视角。快速用鼠标画出整条路径。
- **固定运动轴**：勾选锁定 X / Y / Z，则拖动、点击、键入坐标时该轴坐标被锁住不变，运动只发生在未锁定的轴上。
- 逆解只调整关节角，**绝不改变连杆长度**（固定长度的骨骼始终刚性；结构尺寸只能在 CAD 模式编辑）。
- 关节滑块：手动精调每个关节（旋转关节按度、移动关节按 mm）。
- 摆好一个整机姿态后「＋ 捕获关键帧」，软件在关键帧之间做关节空间平滑插值播放。

**模式切换**
- 顶部醒目的分段开关：`🎬 动作模式` ↔ `🛠 CAD 建模`，一键来回切换。两个模式共用同一机器人模型——在 CAD 里新增/修改的部件，切回动作模式后立即同步出现，可直接拖动控制。

**CAD 工作台（从零搭建任意机器人）**
- 切到「🛠 CAD 建模」进入独立工作台，可「📄 新建空白」从一个底座开始搭建。
- **添加可动部件（骨骼）**：选中父连杆后，一键添加 `旋转关节` / `移动关节` / `固定骨骼` / `末端工具`，自由生长出任意结构（机械臂、多足、龙门…）。
- **大小与外观建模**：每个部件可设连杆形状（圆柱半径 / 方块宽厚）、**颜色**、以及附加几何（方块 / 圆柱 / 球体及其尺寸），自由塑造机器人外观。
- **结构树**：左侧层级树，点选 / 查看每个连杆与关节，显示关节类型与末端标记。
- **操作手柄 Gizmo**：选中连杆后出现三轴箭头（平移）与三环（旋转）手柄，像 CAD 一样精确摆放连杆方向与位置；支持**网格吸附**（可设步长）。
- **数值精确输入**：原点 X/Y/Z、关节限位可直接键入数值。
- **连杆形状与尺寸**：每个连杆可设为圆柱（半径）或方块（宽×厚），真实几何渲染。
- 关节属性：类型（旋转/连续/移动/固定）、轴向、限位、是否末端执行器。
- 「把当前姿态设为初始」固化 home 位姿；「✓ 完成设计」无缝切回轨迹模式，立即拖动你刚设计的机器人做运动轨迹。
- 支持节点姿态偏移（四元数 / URDF 的 rpy），导入 URDF 时朝向也能正确还原。

**代码自动生成**（支持任意 N 关节）
- `Arduino`：N 路伺服（含夹爪机型自动加一路），带插值平滑运动。
- `ROS (Python)`：发布 `trajectory_msgs/JointTrajectory` 的 rospy 节点，关节名取自机型。
- `轨迹 JSON`：通用导出，含每帧的关节值与夹爪状态。

## 机器人定义格式（自定义 JSON）

机器人是一棵节点树，每个节点 = 连接到父节点的关节 + 连杆。角度单位**度**，长度 **mm**，节点须按父索引 < 子索引排列：

```json
{
  "name": "我的机器人",
  "type": "arm",
  "hasGripper": true,
  "home": [0, -20, 40],
  "links": [
    { "name": "base", "parent": -1,
      "joint": { "type": "revolute", "axis": [0,1,0], "origin": [0,0,0], "limit": [-180,180] },
      "geometry": { "type": "cylinder", "size": [50,30] } },
    { "name": "J2", "parent": 0,
      "joint": { "type": "revolute", "axis": [0,0,1], "origin": [0,70,0], "limit": [-110,110] } },
    { "name": "tip", "parent": 1,
      "joint": { "type": "fixed", "origin": [0,150,0] }, "endEffector": true }
  ]
}
```

- `joint.type`：`revolute` / `continuous` / `prismatic` / `fixed`
- `joint.origin`：相对父节点的平移（即上一段连杆长度，mm）
- `joint.axis`：旋转/移动轴（父坐标系）
- `endEffector: true`：该节点可被拖拽并参与逆解

## 项目结构

```
index.html         页面与布局
css/style.css       样式
js/
  kinematics.js     通用机器人树模型 + 正运动学 + CCD 逆解 + 导入导出
  presets.js        内置机型（机械臂 / 四足 / SCARA）
  urdf.js           简化 URDF 解析
  orbit.js          轨道相机控制
  gizmo.js          平移/旋转操作手柄（CAD 句柄）
  robot3d.js        Three.js 渲染任意树 + 多末端拖拽 + Gizmo + 真实连杆几何
  view2d.js         2D 俯视图
  motion.js         运动规划：MoveJ/L/C、梯形速度曲线、三点圆弧、节拍与TCP采样
  trajectory.js     全身姿态关键帧（含段运动属性）与速度曲线播放
  designer.js       结构编辑（加删关节、改属性、拖拽改连杆）
  codegen.js        Arduino / ROS / JSON 代码生成
  examples.js       按机型生成的示例运动
  app.js            主控制器
```

## 技术说明

- 渲染：Three.js (r128, CDN)。逆运动学：CCD（循环坐标下降），按末端所在运动链求解，支持旋转与移动关节，目标超出工作空间自动就近。
- 坐标系 Y 轴向上；内部旋转用弧度，显示/导出为度。
- 已通过离线数值验证：正运动学几何合理、IK 亚毫米收敛、四足足端着地、设计器增删节点索引一致、各机型代码生成关节数正确。

## 适配真实硬件

- 结构尺寸：在设计模式中拖拽，或改 JSON 的 `origin`/`limit`。
- Arduino 引脚：改生成代码顶部 `SERVO_PINS`；伺服零位默认 `servo = 关节角 + 90`。
- ROS：改 `TOPIC` 与 `JOINT_NAMES`。
